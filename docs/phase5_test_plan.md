# Phase 5：前端与监控体验测试方案

## 1. 测试目标
验证 Phase 5 中基于 Chainlit 实现的流式 Web 交互界面以及 LangSmith 可观测性接入是否符合设计预期。确保用户可以通过浏览器流畅地进行单轮查询、多轮追问、实时查看工具抽屉，并验证后台 Trace 数据正常上报。

## 2. 测试环境准备
1. 确保 `.env` 文件中已正确配置 `DEEPSEEK_API_KEY`。
2. 确保 `.env` 文件中已正确配置 LangSmith 相关变量：
   ```env
   LANGCHAIN_TRACING_V2=true
   LANGCHAIN_PROJECT=emmc-copilot
   LANGCHAIN_API_KEY=your_real_key_here
   ```
3. 在终端启动后端服务：
   ```bash
   uv run chainlit run app.py --watch
   ```
4. 在浏览器中打开：`http://localhost:8000`

---

## 3. UI 初始化与基础操作测试

### Case 3.1: 欢迎页加载与预置状态
*   **操作**: 访问 `http://localhost:8000`。
*   **验收标准**:
    *   页面中央显示 `eMMC 协议智能问答助手` 以及 `chainlit.md` 中定义的静态简介与示例问题。
    *   页面右下角有 `eMMC 协议助手已就绪，请输入问题。` 的初始化提示。
    *   点击右上角的 ⚙️（Settings）图标后，默认版本显示为 `5.1`。

### Case 3.2: 规范版本实时切换
*   **操作**: 点击右上角 ⚙️，将 `eMMC 规范版本` 从 `5.1` 切换到 `4.51`，再切换回 `5.1`。
*   **验收标准**:
    *   每次切换后，聊天窗口立即出现系统提示：`已切换至 eMMC 4.51 规范，对话上下文已重置。`（或对应版本）。
    *   无页面白屏，状态平滑过渡。

### Case 3.3: 切换至 "all" 全版本模式
*   **操作**: 点击右上角 ⚙️，将版本切换为 `all`，随后提问：`"HS200 和 HS400 各在哪个版本引入？"`
*   **验收标准**:
    *   切换后提示：`已切换至 eMMC all 规范，对话上下文已重置。`
    *   Agent 检索到的文档片段涵盖多个版本（4.51、5.0、5.1 均有引用），不被单一版本过滤。

## 4. 核心功能与流式输出测试

### Case 4.1: 常规事实检索与流式打印
*   **输入**: `"EXT_CSD[33] CACHE_CTRL 第 0 位的含义是什么？"`
*   **验收标准**:
    *   消息发出后，聊天气泡中首先出现带图标的 `search_emmc_docs` 步骤（Tool Step）。
    *   点开 `search_emmc_docs` 步骤，可以清晰地看到发给大模型的输入参数 `{'query': 'CACHE_CTRL [33] bit 0'}` 以及截断的输出。
    *   当工具执行完毕后，主回答气泡开始**逐字流式打字输出**答案。

### Case 4.2: 复杂专用工具触发（数值计算）
*   **输入**: `"当前设备读取到的 BOOT_SIZE_MULT 寄存器值为 32，请问它的 Boot 分区实际有多大？"`
*   **验收标准**:
    *   出现名为 `calculate` 的工具调用步骤。
    *   点开可看见 input 为两行格式（`_fmt_input` 以换行分隔 key-value）：
        ```
        formula: boot_size
        boot_size_mult: 32
        ```
    *   最终流式回答准确给出：`4096 KB` 或 `4 MB`，没有发生死循环。

### Case 4.3: 并发与耗时工具触发（跨版本比较）
*   **输入**: `"对比 eMMC 4.51、5.0、5.1 的 Command Queuing 支持情况。"`
*   **验收标准**:
    *   出现 `compare_versions` 工具调用步骤。
    *   点开输出可查看到三个版本的检索文档片段拼接。
    *   最终回答完整覆盖三个版本的异同点（如 5.1 明确支持 CQE）。

## 5. 跨轮次上下文关联测试

### Case 5.1: 代词继承与记忆保持
*   **轮次 1 输入**: `"HS200 模式是什么？"`
    *   *期望*: 简短介绍 HS200（单数据速率 200MHz 高速模式）。
*   **轮次 2 输入**: `"那么它和 HS400 从引脚和时钟上有什么区别？"`
    *   *期望*: 模型成功继承"它 = HS200"的上下文，调用工具去检索并对比 HS200 和 HS400。
*   **轮次 3 操作**: 刷新页面（F5），等待出现 `eMMC 协议助手已就绪，请输入问题。` 提示。
    *   *说明*: 使用 F5 刷新而非 Clear Chat 按钮，确保触发 `on_chat_start` 重新生成新的 `thread_id`，彻底隔离上下文。
*   **轮次 4 输入**: `"你还记得我上一个问题吗？"`
    *   *期望*: 模型回答不记得（新 session 的 `MemorySaver` 为空），证明上下文隔离机制生效。

## 6. LangSmith 可观测性链路测试

### Case 6.1: Trace 上报完整性验证
*   **操作**: 完成上述任意一个提问后，登录 [https://smith.langchain.com/](https://smith.langchain.com/)。
*   **验收标准**:
    *   左边栏进入 `Projects` -> 找到项目名称 `emmc-copilot`。
    *   列表中出现刚才的请求执行记录（Traces）。
    *   点击任意记录进入详情瀑布图，能够清晰折叠/展开看到：
        1. Agent Graph 的进入
        2. `ChatOpenAI` 决定调用 Tool 的内部思考过程
        3. 检索工具/纯计算工具的执行耗时（Latency）
        4. 最后一步生成 Token 的过程
    *   在 Trace 详情的 `Metadata` 字段中确认存在 `session_id` 键，其值为当前 Chainlit session 的 ID；同一会话内的多次对话 `session_id` 一致，不同会话之间 `session_id` 不同。

## 7. 错误处理与边界情况测试

### Case 7.1: 版本切换失败时的用户提示
*   **操作**: 临时将 `.env` 中 `DEEPSEEK_API_KEY` 改为无效值，重启服务，点击右上角 ⚙️ 切换版本。
*   **验收标准**:
    *   聊天窗口显示 `版本切换失败：…` 的错误提示，不出现空白卡死。
    *   旧版本的 Agent 仍保留在 session 中（`cl.user_session.get("graph")` 不被覆盖为 None）。
*   **恢复**: 测试完毕后还原 `.env` 正确值。

### Case 7.2: 会话未初始化时的防护提示
*   **操作**: 在浏览器开发者工具的 Console 中执行以下 JS，模拟 `graph` 丢失后直接发送消息：
    ```js
    // 此操作仅用于模拟异常场景，无法直接操作服务端 session
    ```
    实际验证方式：临时注释 `on_start` 中的 `cl.user_session.set("graph", graph)` 一行，重启服务后发送任意消息。
*   **验收标准**:
    *   聊天窗口显示 `会话未初始化，请刷新页面重试。`，不抛出 `AttributeError`。
*   **恢复**: 还原注释。

### Case 7.3: 兜底回答提示（Agent 未生成文本）
*   **说明**: 正常业务流程中 Agent 始终会生成文本，此 Case 为防御性验证。
*   **操作**: 临时在 `on_message` 的事件循环内注释掉 `on_chat_model_stream` 分支，重启服务后发送消息。
*   **验收标准**:
    *   聊天窗口显示 `未能生成有效回答，请重试。`，不出现空气泡。
*   **恢复**: 还原注释。

## 8. 并发初始化测试

### Case 8.1: 双 Tab 同时初始化互不干扰
*   **操作**: 同时打开两个浏览器 Tab，均访问 `http://localhost:8000`，观察各自初始化过程。
*   **验收标准**:
    *   两个 Tab 均出现 `eMMC 协议助手已就绪，请输入问题。` 提示，互不阻塞。
    *   在两个 Tab 中分别提问不同问题，各自答案独立，不出现内容串扰。
    *   在 LangSmith 中可观察到两条 Trace，`metadata.session_id` 值不同。
